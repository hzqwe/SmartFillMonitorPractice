# SmartFillMonitorPractice

## 项目简介

SmartFillMonitorPractice 是一个基于 WPF 的工业监控练习项目，围绕“灌装监控上位机”场景构建，覆盖了登录注册、权限控制、PLC 通信、报警管理、生产记录、日志查询、系统设置和仿真联调等核心能力。  
当前版本已经完成业务规则收口、核心服务接口化、基础测试补齐和最终交付整理，适合用于课程答辩、项目演示、简历展示和后续工程化演进。

## 核心模块说明

### 1. 登录注册与用户管理
- 支持登录、公开注册、首次改密、管理员重置密码
- 角色收束为两类：
  - 管理员
  - 工程师
- 公开注册安全边界：
  - 当系统没有任何用户时，可通过注册创建首个管理员
  - 一旦系统已有用户，公开注册只允许创建工程师

### 2. 主界面与导航
- 主窗口负责页面壳层承载
- Dashboard 负责真实设备监控
- 仿真联调用于 Modbus Slave 联调与演示
- Header 区域统一展示：
  - 当前批次
  - 当前时间
  - PLC 连接状态
  - 状态灯

### 3. PLC 通信
- 基于 NModbus4 + 串口 RTU
- 支持：
  - 自动连接
  - 自动重连
  - Holding Register 读取
  - Coil / Register 写入
  - 条码读取
- 连接状态、读通状态和数据采集状态已做区分

### 4. 报警管理
- 支持活动报警、历史报警、测试报警
- 已实现报警责任追踪：
  - 确认人 ID / 姓名
  - 恢复人 ID / 姓名
  - 触发来源
  - 处理建议
- 历史报警支持查询、筛选、分页

### 5. 生产记录
- 保存生产记录到 SQLite
- 使用幂等键避免重复写入
- 支持查询与 CSV 导出

### 6. 系统日志
- 使用 Serilog 记录运行日志、调试日志和审计日志
- 支持日志查询与导出

### 7. 系统设置
- 支持设备通信参数保存与读取
- 配置文件采用原子写入和损坏备份策略

## 登录 / 注册与角色权限说明

### 登录
- 程序启动后始终先进入登录页
- 登录成功后进入主界面
- 登录失败会给出明确提示
- 连续失败会触发临时锁定

### 注册
- 登录页提供“注册”入口
- 如果系统没有任何用户，可创建首个管理员
- 如果系统已有用户，公开注册只能创建工程师

### 角色权限

#### 管理员
- 系统设置保存
- 用户管理
- PLC 控制
- 报警确认与恢复
- 日志 / 记录导出

#### 工程师
- PLC 控制
- 报警确认与恢复
- 日志 / 记录导出
- 不可进入管理员专属设置能力

## PLC 通信与仿真说明

### 真实设备页
- Dashboard 页面用于真实设备监控
- 只有收到真实 PLC 数据后才更新实时数值
- 图表动画受运行状态控制，不会因为 PLC 在线就自动播放

### 仿真联调页
- 面向 Modbus Slave 调试场景
- 必须真实读通 Modbus Holding Register 后，才允许进入运行态
- 图表基于真实采样值连续追加
- 即使采样值不变，也会持续形成采样推进效果

## 报警确认 / 恢复追责说明

- 报警触发时记录触发来源
- 报警确认时记录：
  - 确认用户 ID
  - 确认用户名
  - 确认时间
  - 处理建议
- 报警恢复时记录：
  - 恢复用户 ID
  - 恢复用户名
  - 恢复时间
- 历史报警可用于责任追溯

## 配置文件说明

- 配置文件由 `IConfigService` 负责读取和保存
- 支持：
  - 默认配置生成
  - 配置校验
  - 原子写入
  - 损坏备份恢复

## 数据库与日志说明

### 数据库
- SQLite
- 默认数据库文件：
  - `SmartFillMonitorPractice.db`

### 主要表
- Users
- AlarmRecord
- ProductionRecords
- SystemLog

### 日志
- 文本日志：`Logs\log-*.txt`
- SQLite 日志表：`SystemLog`

## 如何运行项目

### 1. 构建

```powershell
dotnet build .\SmartFillMonitorPractice.slnx -c Debug --no-restore
```

### 2. 启动

可直接在 Visual Studio 中启动 `SmartFillMonitorPractice` 主项目，或运行输出目录中的可执行文件。

### 3. 首次使用

- 如果系统内没有任何用户：
  - 进入登录页
  - 点击“注册”
  - 创建首个管理员
- 注册成功后返回登录页，输入账号密码登录

## 如何运行测试

```powershell
dotnet test .\SmartFillMonitorPractice\SmartFillMonitorPractice\Tests\SmartFillMonitorPractice.Tests.csproj -c Debug
```

当前测试主要覆盖：
- 用户认证与注册边界
- 报警流转
- 生产记录幂等
- 配置服务
- 壳层稳定性与 PLC 生命周期

## 演示建议流程

适合答辩 / 面试 / 录屏演示的顺序如下：

1. 启动项目，展示登录页
2. 首次无用户时，通过注册创建管理员
3. 使用管理员登录
4. 展示主界面导航、Header 状态、权限入口差异
5. 进入仿真联调页，演示 Modbus Slave 联调
6. 展示 PLC 在线 / 断开 / 重连的状态变化
7. 触发测试报警，演示确认、恢复与责任落库
8. 展示历史报警的确认人、恢复人、处理建议
9. 展示生产记录查询与导出
10. 展示系统日志查询与导出
11. 切换用户，演示管理员与工程师权限差异
12. 退出系统，说明退出清理和资源释放

## 当前已知限制

- NModbus4 的部分底层串口读取不可做到毫秒级取消，极端情况下断开速度仍受串口超时影响
- 当前仍为单机本地 SQLite 架构，暂未扩展到网络多端部署
- 仍存在少量历史命名兼容字段，为了兼容旧库没有一次性全部清除
- 目前测试以核心业务和稳定性回归为主，尚未引入 UI 自动化测试
